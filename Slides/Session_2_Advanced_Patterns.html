Why LangGraph?

LangGraph addresses the limitations of simple ReAct agents by providing graph-based workflows with explicit state management.

Simple ReAct Agent LangGraph
Linear execution Graph-based workflows
Implicit state Explicit state management
Hard to debug Visual, traceable flow
No persistence Built-in checkpointing
Limited control Fine-grained routing

Payments Example: Settlement Workflow

A settlement processing workflow needs:

  • Multiple approval gates (compliance, finance, operations)
  • Conditional routing based on settlement amount
  • State persistence across business days
  • Human-in-the-loop for large settlements

LangGraph handles all of this elegantly with its graph structure.

1. State

A TypedDict that flows through the graph

from typing import TypedDict class SettlementState(TypedDict): transaction_id: str amount: float status: str approvals: list messages: list

2. Nodes

Functions that process and update state

def classify_transaction(state): # Analyze and classify if state["amount"] > 10000: return {"status": "high_value"} return {"status": "standard"}

3. Edges

Connections between nodes (can be conditional)

# Simple edge graph.add_edge("classify", "process") # Conditional edge graph.add_conditional_edges( "classify", route_by_status, # routing function {"high": "review", "low": "auto_approve"} )

4. Checkpoints

Saved state for persistence/resumption

from langgraph.checkpoint.memory import MemorySaver checkpointer = MemorySaver() app = graph.compile(checkpointer=checkpointer) # Resume from checkpoint state = app.get_state(thread_id="txn-123")

from typing import TypedDict from langgraph.graph import StateGraph, END # 1. Define State class SettlementState(TypedDict): transaction_id: str amount: float status: str approval_status: str # 2. Define Nodes def classify(state: SettlementState) -> dict: if state["amount"] > 50000: return {"status": "high_value"} return {"status": "standard"} def require_approval(state: SettlementState) -> dict: # Would integrate with approval system return {"approval_status": "pending_review"} def auto_approve(state: SettlementState) -> dict: return {"approval_status": "approved"} # 3. Routing Function def route_by_value(state: SettlementState) -> str: if state["status"] == "high_value": return "require_approval" return "auto_approve" # 4. Build Graph graph = StateGraph(SettlementState) graph.add_node("classify", classify) graph.add_node("require_approval", require_approval) graph.add_node("auto_approve", auto_approve) graph.set_entry_point("classify") graph.add_conditional_edges("classify", route_by_value) graph.add_edge("require_approval", END) graph.add_edge("auto_approve", END) # 5. Compile and Run app = graph.compile() result = app.invoke({ "transaction_id": "TXN-001", "amount": 75000, "status": "", "approval_status": "" }) # result["approval_status"] == "pending_review"

Other Frameworks Overview

SmolAgents (Hugging Face)

Minimal, code-focused agents with transparency and simplicity.

  • Philosophy: Agents write and execute code directly
  • Best for: Lightweight tasks, learning, prototyping
  • Key feature: Code as the action language
  • Overhead: Very low
from smolagents import CodeAgent, tool @tool def calculate(expression: str) -> float: return eval(expression) agent = CodeAgent(tools=[calculate]) agent.run("What is 15% of 250?")

Google ADK

Production-ready agent framework for Google Cloud ecosystem.

  • Philosophy: Enterprise-grade, scalable deployment
  • Best for: Production on Google Cloud/Vertex AI
  • Key feature: Native GCP integration
  • Overhead: Higher (cloud infrastructure)
from google.adk import Agent agent = Agent( model="gemini-pro", tools=[...], deployment_config={"region": "us-central1"} )

Payments Framework Recommendations

  • Fraud Detection Agent: LangChain (single agent, multiple tools)
  • Compliance Review Team: CrewAI (role-based: researcher, analyst, reporter)
  • Settlement Processing: LangGraph (stateful workflow with approvals)
  • Customer Dispute Resolution: AutoGen (conversational with human-in-loop)
  • Transaction Analytics: SmolAgents (code-based data analysis)
