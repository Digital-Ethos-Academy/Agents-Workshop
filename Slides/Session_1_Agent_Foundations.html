What is an AI Agent?

An AI agent is an LLM-powered system that goes beyond simple question-answering. Unlike a basic chatbot that responds to queries directly, an agent can reason, plan, use tools, and iterate until a task is complete.

Simple LLM Call

  • User sends prompt
  • LLM generates response
  • Single turn, no tools
  • No iteration or planning

AI Agent

  • Reason about tasks and plan steps
  • Act using tools (APIs, databases, code)
  • Observe results and adjust approach
  • Iterate until the task is complete

Payments Industry Example

Consider a Transaction Analysis Agent at a payments company. When asked "Analyze transactions for merchant ID 12345 from last week," it doesn't just generate text - it:

  • Queries the transaction database
  • Calculates statistics and identifies patterns
  • Flags suspicious activity based on rules
  • Generates a detailed report with recommendations

The ReAct Loop

ReAct (Reason + Act) is the core pattern that powers most AI agents. The agent alternates between thinking about what to do and taking actions.

User Query | v +----------+ | REASON | <-- "What should I do to solve this?" +----+-----+ | v +----------+ | ACT | <-- Call a tool (search, calculate, query DB) +----+-----+ | v +----------+ | OBSERVE | <-- "What did the tool return?" +----+-----+ | v Done? --No--> Loop back to REASON | Yes | v Final Response
View ReAct in Action: Fraud Detection Example +
User: "Check if transaction TX-789 is potentially fraudulent" REASON: I need to look up the transaction details first. ACT: get_transaction(id="TX-789") OBSERVE: {amount: $5000, merchant: "Luxury Goods", location: "Nigeria", time: "3:00 AM"} REASON: High amount, unusual merchant category, foreign location, odd time. I should check the customer's transaction history. ACT: get_customer_history(customer_id="C-456") OBSERVE: {avg_transaction: $150, usual_locations: ["US"], typical_merchants: ["Grocery", "Gas"]} REASON: This transaction deviates significantly from normal patterns. Amount is 33x average, new location, new merchant type, unusual hour. I should flag this for review. ACT: flag_for_review(transaction_id="TX-789", risk_score=0.92, reasons=[...]) OBSERVE: {status: "flagged", review_queue: "high_priority"} REASON: Task complete. I've analyzed and flagged the suspicious transaction. Agent: "Transaction TX-789 has been flagged as potentially fraudulent (risk score: 92%). Key concerns: amount 33x above average, unusual location (Nigeria vs typical US), new merchant category, and transaction occurred at 3:00 AM."

When to Use Agents

Good Use Cases

  • Tasks requiring multiple steps with decisions
  • Dynamic tool selection based on context
  • Complex reasoning with external data
  • Tasks where the path isn't predetermined
  • Research and analysis workflows
  • Customer support with system access

Avoid Agents When

  • Workflow is fixed and predictable
  • Simple input -> output transformations
  • Real-time latency is critical
  • You need 100% deterministic behavior
  • Cost per request is a major concern
  • Simple CRUD operations

Payments Decision Framework

At a payments company, consider using agents for:

  • Fraud Investigation: Multi-step analysis of transaction patterns
  • Compliance Review: Gathering evidence from multiple systems
  • Customer Disputes: Researching and resolving chargebacks

Avoid agents for:

  • Transaction Authorization: Needs sub-second latency
  • Balance Lookups: Simple database query, no reasoning needed

Creating Tools with @tool Decorator

from langchain.tools import tool @tool def get_transaction(transaction_id: str) -> dict: """Retrieve transaction details by ID. Args: transaction_id: The unique transaction identifier (e.g., TX-12345) Returns: Transaction details including amount, merchant, and timestamp """ # Your implementation here return db.get_transaction(transaction_id) @tool def flag_for_review(transaction_id: str, risk_score: float) -> str: """Flag a transaction for manual review. Args: transaction_id: Transaction to flag risk_score: Risk score between 0 and 1 """ return f"Transaction {transaction_id} flagged with risk {risk_score}"
View Full Agent Setup Example +
from langchain_openai import ChatOpenAI from langchain.agents import create_react_agent, AgentExecutor from langchain import hub # Initialize the LLM llm = ChatOpenAI(model="gpt-4", temperature=0) # Define your tools tools = [get_transaction, flag_for_review, search_customer_history] # Get a ReAct prompt template from the hub prompt = hub.pull("hwchase17/react") # Create the agent agent = create_react_agent(llm, tools, prompt) # Create the executor (handles the loop) agent_executor = AgentExecutor( agent=agent, tools=tools, verbose=True, # See the ReAct steps max_iterations=10 ) # Run the agent result = agent_executor.invoke({ "input": "Analyze transaction TX-789 for fraud" })

Multi-Agent Systems

Sometimes a single agent isn't enough. Multi-agent systems allow multiple specialized agents to collaborate on complex tasks, each bringing their own expertise.

Collaboration Patterns

Sequential
Agent A --> Agent B --> Agent C

Pipeline processing, each agent adds to the work

Hierarchical
Manager / | \ A B C

Manager delegates to specialized workers

Collaborative
A <--> B \ / v v C

Agents discuss and refine together

Payments Example: Compliance Review Team

A multi-agent system for reviewing suspicious transactions:

Researcher Agent

Gathers transaction data, customer history, merchant info

Analyst Agent

Evaluates patterns, calculates risk scores, identifies anomalies

Compliance Agent

Reviews against regulations, recommends actions, drafts reports

AutoGen vs CrewAI

AutoGen (Microsoft)

Conversational multi-agent framework where agents communicate through messages.

  • Message-passing between agents
  • Strong code execution support
  • Human-in-the-loop built-in
  • GroupChat for multi-agent discussions
assistant = AssistantAgent("analyst") user_proxy = UserProxyAgent("user") user_proxy.initiate_chat(assistant, message="...")

CrewAI

Role-based collaboration where agents have defined roles and work on tasks together.

  • Role/Goal/Backstory for each agent
  • Task-based workflow
  • Intuitive crew composition
  • Built-in delegation
researcher = Agent(role="Researcher", goal="...") task = Task(description="...", agent=researcher) crew = Crew(agents=[...], tasks=[...])
